# Nightly Research — 2026-07-12

## Tonight's focus

## Key findings

### Dan Martell — automation frameworks (cited)
- **The Buyback Principle / Buyback Loop**: use money to buy back time; the loop is Audit → Transfer → Fill, and the **DRIP Matrix** (Delegate / Replace / Invest / Produce) sorts tasks by financial value vs. energy drain to decide what leaves your hands first. ([mindmapai.app](https://mindmapai.app/mind-mapping/buy-back-your-time-by-dan-martell), [agencymastery360.com](https://www.agencymastery360.com/podcast/buy-back-your-time))
- **3-Level AI Agent Framework**: Level 1 = AI assistants (you're still the executor, switching between tools yourself); Level 2 = AI agents (you direct, the agent does the doing, you check it's done); Level 3 = self-managing multi-agent org. Martell's point: most founders stay stuck operating at Level 1 — a faster worker, not a director. ([marketingagent.blog](https://marketingagent.blog/2026/04/09/tutorial-dan-martells-3-level-ai-agent-framework/))
- **What to automate first**: repeatable, low-judgment, high-frequency work — outbound messaging, lead qualification/follow-up, onboarding triggers — freeing humans for judgment calls and relationships. Explicit warning: **optimize the process manually before automating it; automating a broken process just fails faster.** Identify the biggest bottleneck and automate *that* first, before hiring or building anything else. ([shortform.com](https://www.shortform.com/podcast/episode/growth-stacking-show-with-dan-martell-2025-11-29-episode-summary-these-4-ai-automations-will-make-you-1m-with-zero-employees), [mejba.me](https://www.mejba.me/blog/ai-first-company-2026-solo-operator))

### Alex Hormozi — sales/offer frameworks (cited)
- **CLOSER framework** (Clarify → Label → Overview → Sell → Explain → Reinforce) for running the sales call itself. ([salescoachpro.io](https://www.salescoachpro.io/blog/hormozi-closer-framework))
- **AAA objection handling**: Acknowledge → Associate (a similar client's story) → Ask for the sale again — i.e., agree, reframe with proof, re-close. ([jointhefollowup.com](https://www.jointhefollowup.com/p/hormozis-3-a-sales-framework))
- **Value Equation**: Value = (Dream Outcome × Perceived Likelihood) / (Time Delay × Effort). Perceived likelihood (proof, guarantees, testimonials) matters more to the buying decision than actual success rates. ([quantumbyte.ai](https://quantumbyte.ai/articles/alex-hormozi-value-equation-app-monetization), [creatoreconomy.so](https://creatoreconomy.so/p/the-value-equation-for-irresistible-products))
- **Speed to lead**: responding within ~60 seconds vs. hours can lift close rate dramatically — Velocify's research cites a 391% conversion lift responding within one minute; Hormozi cites a real business owner who hits 100% of leads within 60 seconds and closes 55% of them. ([x.com/AlexHormozi](https://x.com/AlexHormozi/status/1879936850959474915), [optif.ai](https://optif.ai/learn/questions/lead-response-time-benchmark/))
- **Core Four lead gen** (warm outreach, free content, cold outreach, paid ads) and **Rule of 100** — pick one lever (100 outreach/day, 100 min content/day, or $100 ad spend/day) and do it daily before adding a new channel. ([shortform.com](https://www.shortform.com/blog/100m-leads-alex-hormozi/))
- **Retention**: ~80% annual retention is Hormozi's B2B benchmark; LTV:CAC target scales with human touch (3:1 no-touch, 6:1 some-touch, 9:1+ high-touch); "client-financed acquisition" = 30-day gross profit ≥ 2× (CAC + COGS). ([blockchain.news](https://blockchain.news/ainews/ai-powered-customer-retention-ascension-model-increase-ltv-and-reduce-cac-with-hormozi-s-strategies), [blog.forecastingperformance.com](https://blog.forecastingperformance.com/p/hormozi-s-favorite-analysis))

### What's actually built in epic-suite (verified by reading the code, not guessing)
- The repo layout has moved on from the `SALES_COCKPIT_SPEC.md`/`supabase/schema.sql` structure the standing brief describes — no such files exist in this checkout. Current structure: root hub `index.html` (role picker + Supabase auth + rank-gating) → `referral/`, `setter/`, `live/` (this **is** the Closer app — "EPIC Closer," an AI calling co-pilot), `success/`, `admin/`, `ledger/` (an unrelated owner-only financial-planning calculator, not part of the sales funnel).
- **EPIC Closer (`live/index.html`, 2316 lines) already implements Hormozi's exact frameworks**: the CLOSER stage tracker (`CLOSER_STAGES`) and an objection bank built as agree → reframe → story (AAA), plus a BANT qualification strip and a "close the deal" panel that recommends a plan/pricing tier live. This is good — it's not a gap, it's validation the build is grounded correctly already.
- **The core gap**: real (non-demo) calls run in a mode the code itself labels `'🎙️ live — no script, track it yourself'` (`live/index.html:1175`). The AI co-pilot (`closerCoach()`) only fires if the rep manually types what the prospect just said into a note box *and* has personally pasted an Anthropic API key into that browser's `localStorage` (`getKey()`, `live/index.html:1603-1611`) — there is no shared/server-side key. A code comment in the same file names the intended end state directly: `"Prototype — simulates the live-listening loop for the team demo. Real version: Twilio audio → Deepgram → Claude → this UI."` (`live/index.html:568`). That pipeline does not exist yet. In Martell's 3-level terms, live-call coaching today sits *below* Level 1 — the rep must both operate and feed the tool mid-call, which is exactly the kind of friction that means it likely doesn't get used on real, high-pressure calls.
- **Fathom integration** (`admin/index.html`) pulls structured data (outcome, objections, next steps) from *finished* Zoom/Fathom recordings — valuable for training/CRM but does nothing for in-call guidance, since it runs after the call ends.
- **No speed-to-lead alerting**: `setter/` and `live/` sort the queue by freshness/score, but a rep only sees a new lead when they open the app and look. There's no push/SMS/email trigger the moment a lead comes in — the single lever Hormozi's research says has the highest, cheapest leverage on close rate (391% within 60 seconds).
- **No retention/churn tracking yet** for Client Success — `admin/index.html:213` has only a comment ("A retention bonus could slot in here"), not a built mechanic. Reasonable to leave for now since it's the secondary priority, not blocking real calls today.

## Concrete next steps for EPIC Suite (ranked by fastest path to more/better real closes today)

1. **Move the Anthropic API key server-side (Supabase Edge Function proxy)** so `closerCoach()` and the AI-prospect practice mode work for every rep out of the box, not only whoever has pasted their own personal key into their browser. This is the cheapest fix on the list — no new infra beyond a pattern already used for Fathom (`admin/index.html`'s `functions.invoke`) — and it's the one that turns an *already-correctly-designed* Hormozi-grounded co-pilot from "works for the person who set it up" into "works for the whole team, on real calls, today." Do this first.
2. **Add a real-time new-lead alert** (SMS or push, piggybacking on existing Supabase functions) the moment a lead lands in `setter`/`live`'s queue. Directly targets the highest-cited leverage point in the research (speed-to-lead). Bounded, single-purpose, not a rebuild.
3. **Scope an MVP for live in-call transcription**, not the full `Twilio → Deepgram → Claude` pipeline in one shot. E.g., start with call recording + auto-populating the note box from the Fathom pipeline already built, before attempting true real-time streaming diarization. This is the highest-value item but also the largest lift — sequence it after #1 and #2, and cap the first version tightly.
4. **Defer** Client Success retention/churn tracking — real, per Hormozi's LTV:CAC framework, but not blocking a single real call from happening this week.
